# OpenTop Public Launch Brief

OpenTop turns messy AI trend signals into ranked, GitHub-ready app ideas with scoring, launch plans, and no-key local mode.

## One-Liner

OpenTop helps AI builders decide which open-source app idea is worth shipping before they spend days writing code.

## Demo Story

- Before: a builder has trend notes, GitHub issues, links, or a product hunch, but no clear wedge for a repository.
- Action: paste the signal into OpenTop, compare generated ideas by pain, urgency, distribution, buildability, and star potential, then select one.
- After: copy a README brief, public launch brief, star growth plan, contributor queue, repo listing pack, or starter repo ZIP.

## Proof Checklist

- [ ] Public demo URL returns HTTP 200 and passes the smoke check.
- [ ] README first screen shows who OpenTop helps, what it produces, current demo status, local proof, and example proof.
- [ ] Screenshot or short recording shows the actual interface and one generated opportunity.
- [ ] Starter issues are current and map to real launch gaps.
- [ ] Launch posts link to GitHub, the hosted or fallback demo, and one concrete generated example.

## Channel Sequence

- GitHub: pin the demo path, public launch brief, starter issues, and repo listing metadata before posting elsewhere.
- Hacker News: lead with the before/after workflow, not a generic AI app claim.
- Reddit: focus on local-first use, transparent scoring, and how the tool avoids requiring a paid API key.
- X/Twitter or newsletter: show one generated opportunity, one score explanation, and one export artifact.

## Follow-Up Loop

- [ ] First 24 hours: answer setup questions and turn repeated confusion into README edits.
- [ ] First 72 hours: open issues for every useful request and close broad requests with scope notes.
- [ ] First week: ship one example, one doc improvement, and one contributor-friendly issue from launch feedback.
- [ ] First month: update the gallery weekly with real AI builder pains and publish a short changelog.

## Feedback Ask

What would make OpenTop more useful for deciding whether an AI app idea is worth building: sharper scoring, clearer examples, more provider presets, or a smaller first-release workflow?

## Current Launch Gate

OpenTop should not be broadly launched until one public demo URL works reliably. Until GitHub Pages is fully restored, use the branch deploy, Vercel, Netlify, Cloudflare Pages Direct Upload, or static ZIP fallback path from the demo fallback guide.
